# 🤝 Moon-Doc 行为准则

> 基于 [Debian 行为准则](https://www.debian.org/code_of_conduct.zh-cn.html)（1.0 版），适配 Moon-Doc 社区。

Moon-Doc 项目（小月亮阅读器）已采纳以下行为准则，对 GitHub 仓库、Issues、Pull Request、Discussions 及其他项目内交流渠道的参与者进行约束。

---

## ✨ 六项核心原则

<table>
<tr>
<td width="6%" align="center">

### 1️⃣

</td>
<td width="20%">

### 🙏 要有礼貌

</td>
<td>

在 Moon-Doc 这种规模的项目中，很难避免遇到与你意见不和、或者难以合作的人。请接受这一事实，并保持礼貌。意见不一致并不是糟糕举止或者人身攻击的借口，让人感觉受到威胁显然不是健康的社区氛围。

</td>
</tr>
<tr>
<td align="center">

### 2️⃣

</td>
<td>

### 💡 善意推定

</td>
<td>

Moon-Doc 的贡献者们可能会通过与您不一样的方式来达成我们对于优秀文档工具的共同目标。请始终假设其他人都在为这一目标而努力。

需要注意的是，我们之中的许多贡献者的母语并不是中文或英语，同时也可能来自不同的文化背景。

</td>
</tr>
<tr>
<td align="center">

### 3️⃣

</td>
<td>

### 🤝 保持合作

</td>
<td>

Moon-Doc 是一个庞大而复杂的项目，这里永远有需要学习的新知识。当你需要帮助的时候，主动向他人求助是一个好的选择。类似地，他人所提供的帮助应该放在改进 Moon-Doc 这一共同目标的上下文中理解。

当你做了有益于项目的事情时，也请不要吝啬向他人解释其原理，这样他人才能够基于你的工作做出进一步的改进。

</td>
</tr>
<tr>
<td align="center">

### 4️⃣

</td>
<td>

### 📝 言简意赅

</td>
<td>

请时刻留心您所写的内容，它会被许多人阅读。书写简短的评论意味着人们可以尽可能高效地理解一次对话。如果必须写出详细的解释，那么请尝试增加一些总结。

请尽量为一次对话带来新的论点，这样每一条评论对于一串讨论来说都是独特的。

请尽量扣紧主题，尤其是在讨论宽泛话题的情况下更需注意。

</td>
</tr>
<tr>
<td align="center">

### 5️⃣

</td>
<td>

### 🌐 保持开放

</td>
<td>

Moon-Doc 社区中采用的大部分通信方式都同时支持公开通信和私人通信。除非是讨论一些敏感话题，对于 Moon-Doc 相关的讨论，应该尽量通过公共渠道（如 GitHub Issues / Discussions）进行。

这一点也适用于 Moon-Doc 相关的求助。这不仅仅是因为在公共渠道寻求帮助更容易得到回复，同时别人的回复中的疏漏和错误也更有可能被大家察觉和纠正。

</td>
</tr>
<tr>
<td align="center">

### 6️⃣

</td>
<td>

### 🛡️ 遇到问题

</td>
<td>

虽然参与者们都应该遵守这一份行为准则，但我们承认人们有时候会遭遇不顺，或者对这份准则中的某一部分不清楚。如果遇到这种情况，您可以尝试回复他们并向其指出这份行为准则。只要符合时宜，这样的交流无论通过公共或者私人渠道均可。然而，不论是通过公共还是私人渠道，消息的内容应当紧扣行为准则中的相关部分。特别地，侮辱和不尊重的言辞应当杜绝。时刻假设别人心怀善意；参与者更有可能并未意识到自己的失当行为，而不是故意降低讨论质量。

严重冒犯他人或屡教不改者，会视情况被 GitHub 组织临时或永久移除。

</td>
</tr>
</table>

---

## 🚨 报告问题

如果您遇到违反行为准则的情况，请通过以下方式联系维护团队：

- 在相关 Issue / PR 中 @ 维护者
- 通过 GitHub 私信联系组织管理员
