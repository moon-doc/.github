## 🌙 小月亮阅读器

> 🌙 我们相信文档应该是**美观的**、**可访问的**、**跨平台的**。
> 小月亮阅读器将原始 Markdown 变成令人愉悦的阅读体验——
> 从浏览器到桌面，从手机到 PDF，一气呵成。

**小月亮阅读器**（Moon-Doc）是一个开源的跨平台文档阅读与渲染引擎，致力于提供**零配置、所见即所得**的文档体验：

| ✨ 特性 | 描述 |
|---------|------|
| 📝 完整 Markdown | markdown-it 全语法 + 表格、脚注、Task List |
| 📐 LaTeX 公式 | `$...$` 行内 & `$$...$$` 块级，KaTeX 服务端渲染 |
| 🎨 代码高亮 | highlight.js 全语言 + 行标记 `{2,4-5}` |
| 🧜 Mermaid 图表 | 浏览器端实时渲染 |
| 🌿 PlantUML | 自动编码为 SVG，无需 Java |
| 📌 Admonition | `:::tip` / `:::warning` / `:::danger` 彩色提示框 |
| 📑 自动 TOC | 从标题生成页面导航 |
| 🔍 全文搜索 | 离线全文检索索引 |
| 📄 PDF 导出 | Puppeteer 所见即所得 |
| 🖥️ 实时预览 | VuePress 风格 UI + 热更新 |

### 🌙 [mdoc](https://github.com/moon-doc/mdoc) — 文档引擎

> **一行命令，精美文档。**

```bash
npx @p-moon/mdoc README.md          # → HTML ✨
npx @p-moon/mdoc README.md --pdf    # → PDF 📄
```

## 🤝 社区准则

小月亮阅读器社区遵循 **[Debian 行为准则](https://www.debian.org/code_of_conduct.zh-cn.html)**，我们相信开放、尊重、协作的力量。

### 🌟 六项核心原则

| # | 原则 | 释义 |
|---|------|------|
| 1 | **要有礼貌** | 意见不一致不是人身攻击的借口，让人感到威胁绝不是健康的社区氛围 |
| 2 | **善意推定** | 始终假设他人都在为共同目标努力；记住许多贡献者母语不是英语，文化背景各异 |
| 3 | **保持合作** | 主动求助，也乐于助人；做出有益的事时请解释原理，让他人能基于你的工作继续改进 |
| 4 | **言简意赅** | 书写简短明了的邮件/评论；尽量为对话带来新论点；扣紧主题 |
| 5 | **保持开放** | 与项目相关的讨论应尽量通过公共渠道进行，这有助于发现和纠正疏漏 |
| 6 | **遇到问题** | 看到失当行为时善意提醒，指向本准则；严重冒犯或屡教不改者可被临时或永久封禁 |

> 📖 完整行为准则详见 [`CODE_OF_CONDUCT.md`](https://github.com/moon-doc/.github-private/blob/main/CODE_OF_CONDUCT.md)

## 🔄 协作流程

<div align="center">

```
  💡 提交 Issue          🍴 Fork & Branch          🔀 Pull Request
  ──────────────  →  ──────────────────  →  ──────────────────
  · Bug / Feature      · feat/fix-xxx          · 关联 Issue
  · 标签分类            · 遵循 Conventional       · 至少 1 人 Review
  · 模板引导            · 单一职责原则              · CI 通过后合并
```

</div>

### 📋 分支与提交规范

| 类型 | 格式 | 示例 |
|------|------|------|
| ✨ 新功能 | `feat: <描述>` | `feat: 添加暗色主题支持` |
| 🐛 修复 | `fix: <描述>` | `fix: 修复 PDF 导出乱码` |
| 📝 文档 | `docs: <描述>` | `docs: 更新 API 文档` |
| 🎨 样式 | `style: <描述>` | `style: 调整侧边栏间距` |
| ♻️ 重构 | `refactor: <描述>` | `refactor: 拆分渲染管道` |
| ⚡ 性能 | `perf: <描述>` | `perf: 优化大文件渲染速度` |
| ✅ 测试 | `test: <描述>` | `test: 补充 KaTeX 渲染用例` |
| 🔧 构建 | `chore: <描述>` | `chore: 升级 Tauri 到 v2.1` |

### 🔐 代码审查

- 所有 PR 须至少 1 人 **Approve** 后方可合并
- CI 必须全部通过（lint + build + test）
- 合并方式：**Squash Merge**（保持主分支历史整洁）
- 重大变更需在 PR 中附带**迁移说明**

> 📖 完整协作规范详见 [`CONTRIBUTING.md`](https://github.com/moon-doc/.github-private/blob/main/CONTRIBUTING.md)
